## 《看房笔记》PWA 视觉与UI规范（补充到 PRD v1.0）

目标气质：**像 Notion 一样的克制、内容优先、结构清晰**。
原则：极简层级、弱装饰、强对齐、清晰留白。

---

# 1. 整体设计原则

### 1.1 核心风格关键词

* Content-first（内容优先）
* Soft UI（轻界面、低对比）
* Structural clarity（结构比装饰重要）
* Calm & Rational（冷静、决策工具感）

### 1.2 禁止事项

* 不使用鲜艳大面积色块
* 不做强拟物图标
* 不用重阴影卡片堆叠
* 不做复杂渐变背景

---

# 2. 布局体系

### 2.1 页面结构（通用）

```
顶部导航（48px）
内容区（16px 页面边距）
模块间距（24px）
模块内部间距（12px）
底部操作区（如有）
```

### 2.2 栅格

* 单栏结构（移动端）
* 最大内容宽度：100%
* 左右安全边距：16px
* 卡片圆角：12px（仅用于“内容块”，不是装饰）

---

# 3. 字体系统

| 用途       | 字号   | 字重  |
| -------- | ---- | --- |
| 页面标题     | 20px | 600 |
| 分区标题     | 16px | 600 |
| 正文       | 14px | 400 |
| 次级信息     | 12px | 400 |
| 数字强调（总分） | 22px | 600 |

字体建议：

* iOS：SF Pro
* Android：Roboto
* Web：system-ui, -apple-system

---

# 4. 色彩系统（Notion风格克制）

| 类型     | 颜色                      |
| ------ | ----------------------- |
| 背景     | #FFFFFF                 |
| 分割线    | #EAEAEA                 |
| 主要文字   | #222222                 |
| 次级文字   | #666666                 |
| 提示文字   | #999999                 |
| 主色（强调） | #2F80ED（仅用于按钮/选中）       |
| 正向标签   | #EAF6EF（底） + #2E7D32（字） |
| 风险标签   | #FDECEA（底） + #C62828（字） |

不使用高饱和渐变。

每个模块是一个“Block”：

* 标题（小号灰字）
* 内容区（输入/按钮）
* 下边分割线

例如：

```
[照片]
客厅  +拍照
卧室  +拍照
...
```

## 5.3 输入控件

### 滑杆（评分）

* 细线滑轨
* 滑块为小圆点
* 当前数值显示在右侧小灰字

### 开关（启用维度）

* iOS 风格 Switch
* 关闭时为浅灰

### 文本输入

* 无边框
* 底部 1px 分割线
* Placeholder 灰色

# 6. 动效与反馈

| 行为      | 动效       |
| ------- | -------- |
| 新建房源卡出现 | 轻微淡入     |
| 切换 Tab  | 水平滑动     |
| 滑杆调整    | 实时数值变化   |
| 权重影响排序  | 列表轻微重排动画 |

不做夸张过渡。

---

# 7. 视觉层级策略（极重要）

1. **信息优先级 = 字重 + 字号，而不是颜色**
2. 分组靠留白而不是框线
3. 一屏只出现 1–2 个视觉焦点（例如“总分”）

---

# 8. 产品气质对标

像：

* Notion（信息块结构）
* Linear（数据感、工具感）
* Readwise Reader（极简内容阅读）

而不是：

* 房产平台（营销感）
* 消费类App（强色彩）

---

# 9. 设计验收标准

* 首页不出现超过 2 种强调色
* 任一页面主按钮 ≤ 1 个
* 录入页不出现多列复杂表格
* 总分始终是最大视觉焦点
